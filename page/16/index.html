多线程

五个状态

  1. 新建状态(new)：线程新建时，进入该状态，Thread thread1 = new Thread();
  2. 就绪状态(Runnable):线程调用start()后，进入该状态，表示该线程已经准备好，等待CPU分配资源使用，但实际使用情况取决于CPU。
  3. 运行状态(Running)：CPU根据其调度算法，选择一个线程开始运行，当选择到该线程，该线程进入运行状态
  4. 阻塞状态(Blocked): 处于运行状态中的线程由于某种原因，暂时放弃对CPU的使用权，停止执行，此时进入阻塞状态，直到其进入到就绪状态，才 有机会再次被CPU调用以进入到运行状态。根据阻塞产生的原因不同，阻塞状态又可以分为三种：
    等待阻塞：运行状态中的线程执行wait()方法，使本线程进入到等待阻塞状态；
    同步阻塞 – 线程在获取synchronized同步锁失败(因为锁被其它线程所占用)，它会进入同步阻塞状态；
    其他阻塞 – 通过调用线程的sleep()或join()或发出了I/O请求时，线程会进入到阻塞状态。当sleep()状态超时、join()等待线程终止或者超时、或者I/O处理完毕时，线程重新转入就绪状态。
  5. 死亡状态（Dead）：线程执行完了或者因异常退出了run()方法，该线程结束生命周期。

yield,join,Interrupt

  1. yield()应该做的是让当前运行线程回到可运行状态，以允许具有相同优先级的其他线程获得运行机会。因此，使用yield()的目的是让相同优先级的线程之间能适当的轮转执行。但是，实际中无法保证yield()达到让步目的，因为让步的线程还有可能被线程调度程序再次选中。
  2. 当我们在某线程中调用另一个线程的join()时，这个方法会挂起调用线程，执行被调用线程，直到被调用线程结束执行，调用线程才会继续执行。

  1. interrupt()通过异常进行交互。
    interrupt() 方法只是改变中断状态而已，它不会中断一个正在运行的线程。这一方法实际完成的是，给受阻塞的线程发出一个中断信号，这样受阻线程就得以退出阻塞的状态。 更确切的说，如果线程被Object.wait, Thread.join和Thread.sleep三种方法之一阻塞，此时调用该线程的interrupt()方法，那么该线程将抛出一个 InterruptedException中断异常（该线程必须事先预备好处理此异常），从而提早地终结被阻塞状态。如果线程没有被阻塞，这时调用 interrupt()将不起作用，直到执行到wait(),sleep(),join()时,才马上会抛出 InterruptedException。

wait和sleep

wait

wait是object类的方法，在当前线程持有对象锁的情况下，暂时放弃锁，并释放资源，等待其他线程调用同一对象的notify()或者notifyAll()。
注意，即使只有一个线程在等待，并且有其它线程调用了notify或者notifyAll方法，等待的线程只是被激活，但是它必须得再次获得锁才能继续往下执行。换言之，即使notify被调用，但只要锁没有被释放，原等待线程因为未获得锁仍然无法继续执行。

结果

可以看出，线程一执行->线程一wait()->线程二执行->线程二notify()->此时线程一并不会接着执行，而是等线程二释放锁，之后线程一获得资源，线程一结束->线程二结束

sleep

sleep()是Thread类的方法，实际上sleep和锁没有关系，仅仅是一定时间内不占用资源。

实际使用上，功能上synchronized是后两者的超集。只有考虑性能时，使用后两者。

synchronized

  1. 修饰实例方法，作用于当前实例加锁，进入同步代码前要获得当前实例的锁
  2. 修饰静态方法，作用于当前类对象加锁，进入同步代码前要获得当前类对象的锁
  3. 修饰代码块，指定加锁对象，对给定对象加锁，进入同步代码库前要获得给定对象的锁。
    实际上是一致的，即synchronized锁的是对象，即｛｝内代码对（）内对象的访问加锁。
    synchronized保证了三大特性

volatile

当一个共享变量被volatile修饰时，它会保证修改的值会立即被更新到主存，当有其他线程需要读取时，它会去内存中读取新值。
一旦一个共享变量（类的成员变量、类的静态成员变量）被volatile修饰之后，那么就具备了两层语义：

  1. 保证了不同线程对这个变量进行操作时的可见性，即一个线程修改了某个变量的值，这新值对其他线程来说是立即可见的。
  2. 禁止进行指令重排序。

volatile保证了可见性和有序性

加锁方式

  1. lock(), 如果获取了锁立即返回，如果别的线程持有锁，当前线程则一直处于休眠状态，直到获取锁
  2. tryLock(), 如果获取了锁立即返回true，如果别的线程正持有锁，立即返回false；
  3. tryLock(long timeout,TimeUnit unit)， 如果获取了锁定立即返回true，如果别的线程正持有锁，会等待参数给定的时间，在等待的过程中，如果获取了锁定，就返回true，如果等待超时，返回false；
  4. lockInterruptibly:如果获取了锁定立即返回，如果没有获取锁定，当前线程处于休眠状态，直到或者锁定，或者当前线程被别的线程中断

《高考恋爱一百天》游戏评测及回忆杂谈

Posted on 2018-08-06 | In 杂谈

游戏评测

初步印象

  其实刚进入游戏的几分钟，我是想退款的，因为对比《三色绘恋》，质量显得差许多，比如：

  1. 好像不适应全屏，也没找到调分辨率的地方，全屏模式左右有黑边；
  2. 对话框不是透明的，遮住了画面的最下方部分；
  3. 场景教学楼楼梯口，墙上写的字太没有真实感了，像p上去的；
  4. 背景音乐有点大，我手动调小了一点；
  5. 部分人物配音不够好

游玩感受

  但实际上，我最后还是游玩了这个游戏，还玩了3（4）个结局，玩了十几个小时，最后甚至有种好的小说电影看完之后那种惘然若失的空洞感，这是为什么呢？
  既然之前玩了另一款游戏《三色绘恋》，不妨做一个对比，其实画面音乐配音立绘这些，《三色绘恋》都胜过了《高考恋爱一百天》，但后者取巧的地方在题材。
  相比于《三色绘恋》的艺术班，故事是四五个事件和两次考试，充斥着大量的心理和感情变化描写；《高考恋爱一百天》就简简单单的高考学习谈恋爱。相比于《三色绘恋》的，恨不得最轰轰烈烈的爱情，最特殊离奇的家庭；《高考恋爱一百天》更多是现实存在的人物。《三色绘恋》纯视觉小说，所以才要塑造的像电视剧一样；《高考恋爱一百天》有养成分支选项等因素，因此更有体验感。

游玩体验

  这游戏有50个结局，大概就是感情乘以成绩的数量吧，我完整的玩了三个。
  第一次玩，对于其中的养成因素不了解，道具使用和时间安排不对，导致感情和成绩都不好，后来由于似乎因为“理想”属性较高，在感情成绩家庭等方面都受挫后，进入了“狂战士”路线，大概就是什么都漠视，只有做题和分数，最后劳累病倒后，我没有选择坚持到底，还是服软打电话求助了妹子，于是回到了正常状态，当然很多东西永远回不来了。由于养成方面不会操作加上装备不好，尽管狂战士阶段学习涨了不少，最后结局还是，考了个普通大学，和名牌大学的女主只是最好的朋友。
  之所以上文说装备不好，只因为这游戏多次通关装备可以继承的，第二次游玩时，装备上了第一次后来有的好装备，养成上轻松了不少。女主是开局就有了，游戏剧情开始就是高考还剩一百天时向本来是好朋友的女主表白被接受，这次大概游戏流程就是和性格呆萌的女主一起谈恋爱，约会的主要方式是一起学习，相互鼓励，最后一起考上了外地的名牌大学。这大概是最简单完美的结局。其实女主呆萌的性格挺惹人喜欢的，由于家庭上母亲比较强势，什么都安排好，女主基本什么都不懂，所以很天真烂漫，傻乎乎的接受主角的好，然后对主角好。
  AVG游戏肯定是双女主，相比起来，简直是官方对另一个的恶意，因为呆萌妹子是开局就有了，因此想选另一个女主，需要先和呆萌妹子分手，这设定简直了。背景设定是从小和主角青梅竹马（大概因此一直喜欢主角），后来因为变故父母离婚，和主角关系也疏远了，最后还是爹不亲亲娘不爱，一个人住在主角家旁的老房子。而这次的游戏流程，是主角被她拿下的过程，通过关心、合作、创造机会、拿下主角父母、故意向主角展现自己软弱的一面等等方法，最后把主角拿下。由于养成上轻车熟路，最后一起考到了名牌大学，还得到主角父母认可。
  看了下其他结局大概包括主角堕落混迹网吧、好兄弟喜欢了女主、狂战士到底最后考到清华得到世界失去你等等，我也不想体验了。

游戏评价

  对人物和环境的评价，“贵在真实”。作者显然是参加过高考谈过恋爱的，所以不会有“皇帝用金锄头耕田”的感觉，主角的家庭是最普通的家庭，高考前关心则乱的父母，叛逆期的主角；母亲大包大揽，所以有呆萌性格的女主；父母离异各自重组，独自生活越坚强也就越需要依靠的另一个女主。这些都给人真实感。至于配角，着墨不多因此尽量脸谱化也很取巧。
  对养成等因素的评价，显然是鼓励多次游玩的，所以装备是能积累的，当“神装”之后，养成很简单，但其过程，感觉并不好。首先是数值策划有问题，中期一直学习，属性可以到溢出，我玩时数学成绩学到超过150分，可能是制作组发现这个问题，所以后期回家学习，属性又会缓慢下降最后值相对正常。其次是学习的过程，就是简单的学习就涨，不学就下降，简单的不合实际。最后值得吐槽的是，主角简直是压力怪，动不动就压力爆炸，而呆萌妹子给的道具是降压神器，可以每天专心学习；而青梅竹马给的是帮助学习的，直接后果是，每天可操作时间，也就是晚自习，基本都在睡觉和娱乐，以恢复体力和降压（而成绩依然涨的飞快）。所以是“每天学习都不累”和“白天高效学习，晚上娱乐休息”两种完全不同的的感受啊。
  至于画面、立绘、音乐、配音、场景等硬性的质量，如同我文章开始所言，基本都劣于《三色绘恋》。

回忆与杂谈

  我想之所以这款游戏在市场反响很大，与主题关系很大，玩过这游戏的玩家，都经历过或即将经历高考，都谈过恋爱或者至少有过喜欢的人对爱情的憧憬，而这款游戏反映了那段时间和经历。
  游戏里学习当然很简单，每天只要学习，分数就能上涨，对于我实际上完全并不是这样，有时候学习反而越学越差，不学却更好，当时自己只能解释量变和质变，现在看来还是方法天赋原因吧。记得当时高中三年，化学课基本不听，最多只是完成布置的作业，基本从未在课题和写作业外投入时间到化学，化学成绩却很好，常常年纪第一，最后高考化学成绩也是。相对的是英语，高考英语是考的最不好的一门，而英语也是我额外投入时间精力最多的一门，然而高三一年，做了无数的题，英语却不进反退。只能说真实世界没有属性值，哪有游戏中，我努力我成功这么简单。
  至于恋爱，我高中时没有谈过恋爱，但也有自己喜欢的人，当然不会是电视电影那种歇斯底里的爱，可能也就是默默地好感而已，现在想来，高中那种环境中，要是谈恋爱，两个人互相鼓励，一起学习，互相帮助，说不定真像游戏里那种一起进步呢。不知道为什么在学校似乎并不允许，但好像也没有红线那么严重，谈恋爱的似乎老师最多也只是不允许，也没什么办法。
  想来我高中生涯学习，高一高二也不甚刻苦，比如高一晚自习都在玩了，回寝室也从不学习，回家也不写作业。但仅仅听讲写作业，成绩当时却还可以，真是想不明白。好像高中生涯学习成绩的曲线，大概就是一个完整的正弦函数曲线，最后只记得，第一次考试是第九名，高考也是第九名了。印象颇深的是参加生物竞赛，当时其实对这个还挺有兴趣的，学了很多最后好像什么奖都没有。后来心灰意冷第二次参见，随便混混最后却拿了个全国二等奖，虽然到现在都不知道有什么用，但其实想到学习生物竞赛的时光，好像快乐居多。当然这些已经和这个游戏没有关系了。
  因为回忆会被自己篡改，对感情的记忆已经不甚清晰，也不知道现在的记忆是不是当时的情感。但现在的记忆已经有些颇为痛苦，也不知当时如何了。这么看来，颇为羡慕这个游戏里主角的感情，一起上学一起回家，互相体贴一起努力。

《三色绘恋》游戏评测

Posted on 2018-08-02 | In 杂谈

游戏性

画面

  游戏画面还不错，色彩很明亮鲜艳，各种图完成度都很高。不过立绘偏少，来来回回就几个，但在有限的立绘中，表情画的很好，通过表情很好的展现了人物状态。场景图太少了，整个故事就发生在教室走廊画室家里等十来个，真是让人觉得有点乏。

音乐

  音乐质量还可以，不少音乐很好听，为此还专门买了OST，但觉得部分音乐出现的频率太高了，尤其是《心底的颜色_Orange》这首，简直听腻了，好像没有通过音乐表现来区分两个女主。全程都有配音，配音质量我觉得很好，在俏皮和日常时很真实，在情感爆发时也不会显得浮夸。

剧情

  虽说是galgame，但游戏并没有分支选项，全程都是敲空格，只是在每章结尾有个投票环节，用来最后解锁两个不同的大结局章节。本想看看两个结局分别长什么样，但玩完一个已经要坚持才能看到结尾了，玩不动了。煽情可能比较失败，但有些较有趣的情节，还是给我带来了不少乐趣的。

其他

  毫无意义的把其中一个女主的胸画的特别大，也许是为了噱头或者吸引玩家，导致本来和色情毫无关系的游戏，在steam被打上了色情游戏的标签，可能有助于销量吧，但我觉得有点low。

文学性

总体评价

  百万字的文本，当然可以讨论文学性。
  要是总体评价，给我的感觉就是，“文笔非常好的高中生”。
  可能受日本文化影响较多，语言上总有一种日本动漫的风采，什么“执念”啊，“守护”啊，“梦想”啊，喜欢从各种人口中讲大道理，”坚持努力，找到自我“之类，最后用嘴遁说服人拯救世界。而其实和愚公移山一样，最后移走山的也不是愚公，而是感动天帝而已。也许本来galgame和日本动漫相近，所以总给人一种日本动漫的感觉。

环境

  人物的行为和语言基本是脱离环境的，这句话的意思是：设定是2005年武汉高中生，但以其极低的耦合度，导致其实换个背景基本也没有问题。大部分时候人物说话的方式差不多，并且不像高中生，最大的问题是，不像对话。

描写

  作者非常喜欢心理描写，经常大段的心理描写，并且总是使用比喻的手法，而正是这些占了大量的文本量。实际上，我觉得这样的方法很缺乏表现力，”悲伤像藤曼一样牢牢得捆住我的内心，吸取着它的养分“，这样的句子，也许作者觉得很好， 但我认为心理感受靠比喻的描写这种方式，基本无法让读者感受到其心理状态。”今已亭亭如盖矣“，作者可能需要学习下这句。

人物

  人物塑造，性格的来源必有其背景的原因，这点应该是该作品中人物塑造的优点。几个配角相对好一点，但脸谱化。而主角的塑造就比较失败，玩家操控的角色，失败的原因是，作者试图通过心理描写来体现人物，但如前文所说，这样的心理描写，表现力很差。而两个女主，可能是因为我不懂女生，我根本不知道她们在哭什么，她们在对不起什么。主角是两个女主一切行为的出发点和落脚点，导致人物过于苍白，可能只有最后，日常活动中文芷大小姐吐槽男主的时候，能让我觉得她是真实存在的人。

SpringMVC

  • @RestController = @RespondBody + @Controller

  • @ResponseBody 注解表示该方法的返回的结果直接写入HTTP 响应正文（ResponseBody）中，一般在异步获取数据时使用。
    通常在使用 @RequestMapping 后，返回值通常解析为跳转路径，加上 @Responsebody 后返回结果不会被解析为跳转路径，而是直接写入HTTP 响应正文中。
